### Q1. YAML 是什么？我该如何学习它？

#### YAML 是什么（人话）

**YAML** 是一种写 **配置文件** 的文本格式，名字来自 “YAML Ain't Markup Language”。

你可以把它理解成：

> **用缩进和简单符号，把「名称、版本、输入输出」等信息写成人类能读、机器也能读的结构化文件。**

在本项目里，每个 Agent 的「说明书」就是 **`agent.yaml`**，放在 `社区demo/agents/某个名字/agent.yaml`。

#### 长什么样

用 **缩进** 表示层级（有点像 Python），不用很多 `{}`：

```yaml
id: scrna-full-analysis
name: 单细胞全流程分析
version: 1.0.0
domain: scrna-seq

skill_chain:
  - skill: scrna-qc-filter
    name: 单细胞质控过滤
  - skill: scrna-cluster
    input_from: scrna-qc-filter.filtered_matrix
```

- `id:`、`name:` 这种叫 **键值对**
- 下面缩进的是 **子内容**
- `-` 开头的是 **列表里的一项**

#### 和 JSON 的区别（先知道即可）

| | YAML | JSON |
|---|------|------|
| 适合谁写 | 人手写、手改 | 程序互传 |
| 样子 | 缩进、少标点 | 大括号、引号多 |
| 在本项目 | Agent 源数据 `agent.yaml` | 网页读的 `agents.json` |

同一件事两种写法：

**YAML：**
```yaml
name: 单细胞分析
version: 1.0.0
```

**JSON：**
```json
{
  "name": "单细胞分析",
  "version": "1.0.0"
}
```

#### 在本项目里干什么用

| 文件 | 作用 |
|------|------|
| `agent.yaml` | 描述一个 Agent：叫什么、干什么、引用了哪些 Skill |
| （Skill 组负责）`skill.yaml` | 描述一个 Skill——**不是我们组写的** |

你改 Agent 信息 → 改 **yaml**；网页展示 → 读 **json**（见 Q2）。

#### 学习 YAML 要掌握什么（最小集）

| 优先级 | 学什么 | 学到什么程度 |
|--------|--------|--------------|
| ⭐⭐⭐ | 键值对 `key: value` | 能读懂 |
| ⭐⭐⭐ | 缩进表示层级 | 知道空格数错了会报错 |
| ⭐⭐⭐ | 列表 `- item` | 能看懂 skill_chain |
| ⭐⭐ | 嵌套结构 | 能照着模板改 |
| ⭐ | 高级语法 | 以后用到再学 |

#### 推荐学习资源（国内）

| 类型 | 去哪里 | 说明 |
|------|--------|------|
| 文档 | [菜鸟教程 YAML](https://www.runoob.com/w3cnote/yaml-intro.html) | 1–2 小时通读 |
| 实践 | 改 `社区demo/agents/scrna-full-analysis/agent.yaml` | 改一行 → 运行 export → 刷新网页 |
| 校验 | 搜索「YAML 在线校验」 | 检查缩进是否写错 |

#### 学习路径（建议 1 天）

```
1. 读菜鸟教程前 3 节（语法、键值、列表）     ← 约 30 分钟
2. 打开本项目 agent.yaml，对照教程认字段      ← 约 20 分钟
3. 改 name 或 description，export 后看网页  ← 约 20 分钟
4. 故意少写一行缩进，看会不会 export 报错     ← 约 10 分钟
```

#### 注意事项

- **用空格缩进**，一般 2 个空格一层；不要用 Tab（容易乱）
- 中文内容一般不用引号；有特殊字符时可加引号
- 改完 yaml 记得运行 `export_json.py`，网页才会更新

### Q2. 为什么要从 YAML 导出 JSON？

#### 人话结论

> **yaml 给人改，json 给网页读。**  
> 导出 = 把 Agent 说明书「翻译」成浏览器更方便用的格式。

#### 为什么不能网页直接读 YAML

| 原因 | 说明 |
|------|------|
| 浏览器不原生支持 | 前端 JS 读 JSON 一行 `fetch` + `res.json()` 就行；YAML 还要额外库 |
| 源文件可能有多份 | `agents/` 下每个 Agent 一个 yaml，网页希望 **一个文件** 拉列表 |
| 分工清晰 | 你维护 **agents/** 下的 yaml；**web/data/agents.json** 自动生成，不手改 |

#### 在本项目里的数据流

```
agents/scrna-full-analysis/agent.yaml   ← 你编辑（源）
agents/proteomics-diff-analysis/agent.yaml
              ↓
    scripts/export_json.py              ← 导出脚本（翻译）
              ↓
    data/agents.json                    ← 备份 / 给别的程序用
    web/data/agents.json                ← 网页 fetch 这个文件
              ↓
    web/index.html 列表页 / 详情页       ← 用户看到的
```

#### 你要做的实际操作

```powershell
python "d:\智能体社区\社区demo\scripts\export_json.py"
```

改 yaml → 跑这条命令 → 刷新 http://localhost:8080

#### 能不能不要导出，只用 JSON

可以，但 **不推荐**：

- 手改 JSON 容易漏逗号、括号，不如 yaml 好写
- 以后 Agent 多了，一个 json 里堆几十条很难维护
- 方案里定的是 **Agent 以 yaml 为登记格式**

Demo 阶段：**yaml 是源，json 是网页用的副本。**

#### 和「入库管理」的关系

| 步骤 | 对应 |
|------|------|
| 入库 | 往 `agents/` 里加 / 改 `agent.yaml` |
| 导出 | 运行 `export_json.py` |
| 看得见 | 网页读 `web/data/agents.json` 展示 |
