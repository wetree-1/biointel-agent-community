# 智能体社区 · 任务实施方案

> **平台**：https://biointelos.xdataspeak.com/#/login  
> **版本**：v1.2 | **日期**：2026-05-30  
> **适用**：项目团队内部 + 与平台对接人讨论  
> **实现以 [`社区demo/`](../社区demo/) 为准** · 日常执行见 [执行清单.md](./执行清单.md)

---

## 分工边界（重要）

| 负责方 | 负责内容 |
|--------|----------|
| **本小组** | Agent 入库管理、Agent 开发规范（SOP）、Agent 展示页面 |
| **Skills 小组** | Skill 开发、Skill 注册表、`skill.yaml` 规范（**本组不负责**） |
| **平台方** | BioIntelOS 对接、现有 Agent 家底、页面嵌入方式 |

本小组在 Agent 的 `agent.yaml` 中通过 **`skill_chain` 引用** Skills 组提供的 Skill，详情页展示调用链，但 **不开发、不维护 Skill**。

---

## 一、项目是什么

在 BioIntelOS 平台上，建设 **Agent 能力管理体系**——类似生命科学的「Agent 应用商店 + 开发规范」：

- **Agent**：面向具体场景的完整分析方案（从输入到报告）
- **Skill**：Agent 调用的原子能力，由 **Skills 小组** 开发与注册；本组仅在 agent.yaml 中引用
- **SOP**：Agent 登记与上架的开发规范

让平台 Agent **管起来、有标准、能迭代、看得见**。

---

## 二、项目任务

| 任务 | 说明 |
|------|------|
| **对内管理** | 统一登记平台 Agent，可查版本、能力、负责人 |
| **定标准** | 规定 Agent 怎么描述、怎么测试、怎么上架 |
| **做试点** | 登记 1 个 scRNA-seq 标杆 Agent（skill_chain 引用 Skills 组 Skill） |
| **能展示** | 页面上能浏览 Agent 列表、查看详情和调用链 |
| **对接平台** | 与 BioIntelOS、Skills 组对齐分工与接入方式 |

**现阶段不做**：Skill 开发、Skill 注册表、开放社区、众包上传、激励体系。

---

## 三、最终效果

### 3.1 近期（第一阶段完成后）

- 用户打开页面，能 **浏览、搜索** 已登记 Agent
- 点进详情，能看到 **能力说明、引用了哪些 Skill（skill_chain）、版本信息**
- 仓库里有 **统一格式的 agent.yaml**
- scRNA-seq 标杆 Agent **完成登记**，skill_chain 与 Skills 组对齐
- 新 Agent 可按 SOP 和模板登记，不再各自为政

**对平台团队**：每个 Agent 有清晰档案（版本、负责人、更新记录），知道平台有多少 Agent、该升级哪个。

---

### 3.2 远期（完整形态）— 三方视角

#### 使用者视角（分析师、研究员）

```
我需要做单细胞分析
    → 打开平台，找到「单细胞全流程 Agent」
    → 一键使用，自动跑完
    → 不满意可以提意见
```

分析能力 **找得到、信得过、用得上**。（Skill 的浏览与安装由 Skills 组能力体系支持。）

#### 开发者视角（Agent 贡献者 — 本组规范所面向的角色）

```
我有新的分析场景要做
    → 查 Agent SOP，知道怎么登记
    → 从 Skills 组选用 Skill，写入 skill_chain
    → 提交 agent.yaml，审核通过
    → 上架，用户在 Agent 目录可见
```

#### 平台视角（运营方、管理层）

```
Agent 生态（本组）
    ├── 统一 Agent 注册表（所有 Agent 有目录）
    ├── 统一 Agent 规范（登记、测试、上架有标准）
    └── 持续进化（用户反馈 → Agent 配置优化 → 新版本）

Skill 生态（Skills 组）
    └── 统一 Skill 注册表与开发规范
```

平台从「功能堆叠」变成 **可管理、可扩展、可运营的能力体系**。

## 四、实现总路线

```
第 1 步：定规矩        Agent 怎么描述、怎么登记、怎么上架
    ↓
第 2 步：建目录        所有 Agent 登记进 Git 仓库
    ↓
第 3 步：做样板        登记 scRNA-seq 标杆 Agent（引用 Skills 组 Skill）
    ↓
第 4 步：能看得见      页面上能浏览 Agent、点详情
    ↓
（之后）开放 Agent 贡献、更多领域
```

**第一阶段目标**：四步全部走通，能向平台对接人 **完整演示**。

## 五、第 1 步：定规矩

### 5.1 解决什么问题

没有统一标准时，每个 Agent 写法不同，难维护、难交接。  
本步为 Agent 制定 **「身份证模板（agent.yaml）」** 和 **「登记规范（SOP）」**。

### 5.2 要定的两类东西

#### （1）Agent 元数据标准

**Agent 说明书 `agent.yaml` 必填字段：**

| 字段 | 含义 | 示例 |
|------|------|------|
| `id` / `name` / `version` | 基本信息 | 单细胞全流程分析 v1.0.0 |
| `domain` | 领域 | scrna-seq |
| `scenario` | 适用场景 | 新样本 FASTQ → 细胞注释 |
| `capabilities` | 能力列表 | 质控、聚类、注释 |
| `skill_chain` | 引用的 Skill 及顺序 | 见下方示例 |
| `io` | 整体输入输出 | FASTQ in → h5ad + 报告 out |
| `evaluation` | 验收指标 | annotation_confidence ≥ 0.7 |
| `author` | 维护人 | platform-team |

**skill_chain 示例（引用 Skills 组，非本组开发）：**

```yaml
skill_chain:
  - skill: scrna-qc-filter
    params: { mito_threshold: 0.2 }
  - skill: scrna-cluster
    input_from: scrna-qc-filter.filtered_matrix
  - skill: scrna-annotate
    input_from: scrna-cluster.cluster_labels
```

#### （2）Agent 开发规范 SOP

| 章节 | 内容 |
|------|------|
| 需求调研 | 给谁用、解决什么、不做什么 |
| 如何选择 Skill | 从 Skills 组注册表选用，写入 skill_chain |
| agent.yaml 编写 | 字段说明与示例 |
| 测试要求 | 元数据校验 + 与 Skills 组联调 |
| 验收与上架 | 什么叫合格、什么不能上架 |
| 版本管理 | SemVer、changelog |

### 5.3 执行动作

1. 与平台、Skills 组开会，对齐 Agent 定义及边界
2. 编写 `agent.yaml` 模板（标注必填 / 选填）
3. 编写 Agent SOP v1（2–5 页）
4. 附 1 个 mock 示例
5. 形成会议纪要

### 5.4 产出与验收

| 产出 | 验收标准 |
|------|----------|
| `agent.yaml` 模板 | 含 skill_chain 示例 |
| Agent SOP v1 | 能指导完成一个 Agent 登记 |
| 会议纪要 | 定义、边界、分工已确认 |

**过关标志**：按模板填写 agent.yaml，团队和平台都能理解且不歧义。

## 六、第 2 步：建目录

### 6.1 解决什么问题

Agent 散落各处，无法统一查询和管理。  
本步建立 **Agent 户口系统**——所有 Agent 登记在同一 Git 仓库，格式一致、可校验。

### 6.2 仓库结构

```
agent-community/
├── agents/                     # 所有 Agent
│   └── scrna-full-analysis/
│       └── agent.yaml
├── pipelines/                  # 领域场景说明（可选）
│   └── scrna-seq/
│       └── README.md
├── schemas/
│   └── agent.schema.json       # 仅 Agent 校验
├── scripts/
│   └── validate.py
└── docs/
    └── sop/
```

> 不含 `skills/`——Skill 在 Skills 组仓库。

### 6.3 执行动作

1. 在 GitHub / Gitee 创建 `agent-community` 仓库
2. 初始化目录，提交 agent 模板和 Schema
3. 实现校验：`python scripts/validate.py agents/`
4. 向平台索取 **现有 Agent 清单**
5. 登记第一批 Agent（至少 1 个 mock）

### 6.4 产出与验收

| 产出 | 验收标准 |
|------|----------|
| Git 仓库 | 结构清晰，README 说明分工边界 |
| Schema 校验 | agent.yaml 缺字段能报错 |
| Agent 盘点表 | 平台现有 Agent 数量清楚 |
| 首批登记 | ≥1 个 Agent 符合模板 |

**过关标志**：新加一个 Agent，按模板写入仓库，校验能通过。

---

## 七、第 3 步：做样板

### 7.1 解决什么问题

规矩和目录有了，还需 **登记一个完整的标杆 Agent**。  
本步以 **scRNA-seq** 为试点，完成样板 agent.yaml，skill_chain 引用 Skills 组 Skill。

### 7.2 场景说明

```
输入：FASTQ 或 count 矩阵
    ↓
Step 1 质控（Skill: scrna-qc-filter）
    ↓
Step 2 聚类（Skill: scrna-cluster）
    ↓
Step 3 注释（Skill: scrna-annotate）
    ↓
输出：annotated.h5ad + 报告
```

写入 `pipelines/scrna-seq/README.md` 作为场景说明。

### 7.3 本组做什么 / Skills 组做什么

| 负责方 | 内容 |
|--------|------|
| **本组** | `agents/scrna-full-analysis/agent.yaml`（scenario、io、evaluation、skill_chain） |
| **Skills 组** | skill_chain 中各 Skill 的开发与注册 |
| **联调** | 确认 skill_chain 中的 Skill id 在 Skills 组注册表中存在 |

### 7.4 测试

| 测试类型 | 内容 |
|----------|------|
| 元数据测试 | agent.yaml 通过 Schema 校验 |
| 引用检查 | skill_chain 中的 Skill id 与 Skills 组一致 |
| 执行测试（可选） | 由 Skills 组或平台 demo 实际跑通 |

### 7.5 产出与验收

| 产出 | 验收标准 |
|------|----------|
| 场景说明文档 | 步骤、输入输出清晰 |
| 1 个 agent.yaml | skill_chain 正确引用 Skills 组 Skill |
| 与 Skills 组对齐记录 | Skill id 清单一致 |

**过关标志**：他人看 agent.yaml 能懂场景和引用了哪些 Skill。

---

## 八、第 4 步：能看得见

### 8.1 解决什么问题

前三步在仓库里，用户看不到。  
本步把登记好的 Agent **摆到货架上**——能浏览、能点详情。

### 8.2 页面需求

**列表页：** Agent 名称、领域、版本、描述、搜索（可选）

**详情页：** 场景说明、**skill_chain（展示引用了哪些 Skill）**、输入输出、版本

### 8.3 数据链路

```
Git 仓库（agent.yaml）
        ↓
export_json.py → agents.json
        ↓
前端页面展示
```

**API（后续）：**

```
GET /agents           → Agent 列表
GET /agents/:id       → Agent 详情（含 skill_chain）
```

### 8.4 执行动作

1. 与平台确认：独立站还是嵌入 BioIntelOS
2. 定 JSON 格式，与 agent.yaml 字段对齐
3. 写转换脚本 + 前端列表页 / 详情页
4. 部署到云端

### 8.5 产出与验收

| 产出 | 验收标准 |
|------|----------|
| 列表页 | 展示至少包含样板 Agent |
| 详情页 | 含 skill_chain、版本、输入输出 |
| 部署地址 | 对接人可访问 |

**过关标志**：打开链接，能看 Agent 列表并点进 scRNA-seq 样板详情。
